# Test Flows

This repo has two reference paths right now.

## Reference Methods

### Manual/Synthetic Reference

`tests/fixtures/reference_baking.py`

This method creates expected normal maps and ground-truth arrays for known test
cases. For example, the pedestal rib case manually computes the expected raised
rib normal because we know the intended shape and UV convention.

Use this as an oracle for:

- locking sign conventions,
- checking tangent-space normal encoding,
- preserving expected behavior for curated examples.

### CAD-Derived Reference

`tests/fixtures/cad_reference_baking.py`

This method samples a low CAD face, projects each sample to high CAD faces, gets
the high-face UV and normal from OpenCascade, and converts the high normal into
the low-face tangent space.

Current limitations are marked with FIXME comments in the module:

- nearest-surface projection only,
- approximate native-UV tangent basis,
- one image per low face instead of a packed atlas.

Use this as a scaffold for testing behavior on arbitrary CAD shapes.

## Current Comparisons

### Manual Reference vs CAD-Derived Reference

`tests/python/test_reference_method_comparison.py`

This compares the manual/synthetic reference with the CAD-derived reference on
existing examples.

Current results:

- `flat_to_u_groove`: normals match exactly.
- `flat_to_v_groove`: normals match exactly.
- `pedestal_ribs`: most samples are close, but nearest-surface projection creates
  rib-edge outliers.

These tests compare tangent-space normals today, but non-visual tests can also
compare mapping arrays directly:

- `high_face_id`
- `high_uv`
- `position`
- `distance`
- `hit_mask`
- `status`

Direct UV/mapping comparisons are useful when normal differences are caused by
tangent-basis conventions rather than hit selection.

### C++ Method vs Reference

Planned flow:

1. Generate or load a low/high CAD pair.
2. Run the Python CAD-derived reference method.
3. Run the C++/native mapper on the same input samples.
4. Compare mapping arrays first:
   - status / hit mask,
   - high face id,
   - high UV,
   - hit position,
   - distance.
5. Compare normals after the mapping comparison is understood.

For new shapes, add them first as deterministic fixtures in
`tests/fixtures/cad_cases.py`, then compare both the reference and native paths
against those fixtures.
